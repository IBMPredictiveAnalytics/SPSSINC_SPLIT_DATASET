Создайте набор файлов SAV, расщепив активный набор данных

Эта процедура работает аналогично команде SPLIT FILES, но записывает расщепления в новые файлы SAV, а не определяет группы для отдельной обработки процедурами. Это особенно полезно, когда блок синтаксиса преобразования и/или процедуры нужно применить к каждой группе отдельно. Команду расширения SPSSINC PROCESS FILES или эквивалентное диалоговое окно Утилиты>Обработать файлы можно использовать с выходными данными этой процедуры без ограничений метода SPLIT FILES.

Разделить наблюдения по. Выберите одну или несколько переменных, которые будут использоваться для расщепления активных данных в набор файлов. Переменные могут быть строковыми или числовыми. Числовые переменные должны иметь целые значения. В диалоговом окне не выводятся переменные с уровнем шкалы измерений.

Положение вывода. Файлы вывода можно записать в заданный каталог, который при необходимости будет создан, а также можно применить команду для создания нового временного каталога и записывать туда файлы. Включите соответствующую радиокнопку. В случае задания каталога выберите его в поле Каталог файлов вывода. Если выбрано использование временного каталога, этот управляющий элемент должен быть пустым.

• В конце задания временный каталог не удаляется.

Можно задать паттерн, использующий значения или метки переменных расщепления, чтобы направить файлы вывода в каталоги на основе значений или меток расщепления. Для ссылки на переменные расщепления указывайте их как ${variable имя} в спецификации каталогов. Например, при расщеплении по переменной xvar, у которой есть значения "a" и "b", запишите
c:/target/x${xvar}
При этом в зависимости от значения xvar файлы будут записываться в /target/xa или в /target/xb.

Можно использовать любое количество ссылок на переменные. Каталоги будут создаваться при необходимости. Они создаются только для действительно существующих значений, обнаруженных среди данных. Механизм паттерна можно использовать также для имен файлов.

• Символы, недопустимые в именах каталогов или файлов, например, ("), будут заменены на символ подчеркивания (_).

• Результаты могут отличаться для файловых систем, в которых имена файлов и каталогов регистрозависимы или же регистронезависимы (например, Windows).

Удалите существующие файлы SAV из каталога назначения. Если используется непосредственно заданный каталог назначения, все существующие файлы SAV можно удалить из него до создания новых файлов. Это сокращает трудоемкость и упрощает последующую обработку файлов группой, но убедитесь, что вы понимаете, что именно вы удаляете. При использовании паттернов каталоги очищаются только для значений паттерна, которые действительно встречаются в наборе данных.

Имена выходных файлов. По умолчанию имена выходных файлов образуются из сочетания значений или меток всех переменных расщепления, разделенных символом подчеркивания. Это правило можно изменить, определив фиксированное имя или паттерн аналогично именам каталогов, как описано выше. Расширение .sav будет добавлено автоматически.

• Фиксированное имя файла должно включать в себя паттерн имени каталога, иначе все данные будут записаны в один файл.

Файл перечисления вывода. Введите спецификацию файла, чтобы создать файл, в котором будут перечисляться все созданные этой командой файлы SAV. Эта спецификация содержит полный путь к файлу и значение расщепления для этого файла. Спецификация упрощает использование выходных данных этой процедуры при последовательной обработке набора файлов или для ведения журналов. Файл записывается в кодировке Unicode utf-8.

Параметры

Используйте диалоговое подокно Опции для задания базы именования, а также для указания, выводить ли список файлов в программе просмотра

Имена файлов вывода. Имена файлов можно сгенерировать тремя способами:

• По значениям переменных расщепления. Например, с двумя переменными у имен файлов будет вид значение1_значение2.sav. Для системного значение отсутствия корневым именем будет $Sysmis. Для строковых переменных пустое значение приводит к файлу с именем .sav. Это разрешенное имя файла, но в некоторых операционных системах оно не появляется в перечислении файлов каталога по умолчанию. Если используется паттерн, значения переменной используются для подстановки в него.

• По меткам значений переменных расщепления. Метки значений для переменных расщепления используются как корневое имя. Если метки нет, вместо нее используется само значение. Если применяется паттерн, метки значений используются для подстановки в него.

• По последовательному номеру. Корневые имена файла изменяются от 0001 до максимального нужного номера. Эта опция полезна, если значения меток могут привести к неудобочитаемым именам. Если задано имя файла, эту опцию использовать нельзя.

Все символы, которые могут быть недопустимыми в имени файла для некоторой операционной системы, заменяются пробелами. Эти символы включают в себя ", /, \, <, >, * и ?.

Префикс имени. Включите этот переключатель, чтобы включить текст ниже как начало каждого корневого имени. Оставшаяся часть имени будет определяться параметрами имен файлов вывода, разделенными нижним подчеркиванием.

• Эту опцию нельзя использовать с паттерном.

Вывести список записанных файлов. Включите этот переключатель, чтобы создать в программе просмотра файлов список записанных файлов и связанных значений переменных расщепления.

Дополнительные возможности

Введите команду
SPSSINC SPLIT DATASET /HELP.
чтобы вывести полное описание синтаксиса. Используя синтаксис, выходные данные можно записать в текущий рабочий каталог обработчика SPSS. Имя этого каталога выводится командой SHOW DIRECTORY.

Количество передач данных, требуемых для создания файлов SAV, зависит от вашей версии IBM SPSS Statistics. Для версии 17 при одной передаче может быть создано десять файлов SAV. В более новых версиях одновременно можно создать 64 файла.

Технические требования

Этой команде требуется подключаемый модуль программирования Python. Его можно найти на установочном компакт-диске IBM SPSS Statistics или скачать с сайта SPSS Community (www.ibm.com/developerworks/spssdevcentral).
